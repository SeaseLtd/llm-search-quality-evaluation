## How to Contribute

There are several ways you can help:

* **Report Issues:** If you find a bug/issue, please create a [github issue](https://github.com/SeaseLtd/llm-search-quality-evaluation/issues).
* **Suggest Features:** Have an idea for a new metric or evaluation method? Open an issue to discuss it.
* **Contribute To Existing Issues:** Tackle [open github issues](https://github.com/SeaseLtd/llm-search-quality-evaluation/issues)  or submit new features.

## Existing Issues

We have a list of open issues that we need help with. This is the best place to start if you are looking for code contributions.

* **[View Open Issues](https://github.com/SeaseLtd/llm-search-quality-evaluation/issues)**

If you find an issue you would like to work on, please comment on it and so that to avoid duplicating work.

## Pull Request Process

We use the standard GitHub "Fork & Pull" workflow.
Fork our repo and once you have changes, commit them to your fork, then create a Pull Request (PR) to our repo.

Before submitting a PR, please ensure:
* [ ] Unit test(s) is/are added and all tests do pass
* [ ] If you added a new feature, you have updated the README or relevant documentation.

## Reporting Issues

This section guides you through submitting a bug report.

When opening a new issue, please include:
1.  **A clear title.** Descriptive, self-explanatory title
2.  **Description:** What the issue is about ?
3.  **Suggested approach:** (Optional) Any suggested idea to solve the problem if you have. 


**Questions?**
If you have any questions, feel free to open a discussion in the Issues tab.
